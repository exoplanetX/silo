## 1. Background

SILO now has a native branch-and-bound Python API for small MIPs:

```python
from silo.io.json_reader import read_json_model
from silo.mip.branch_and_bound import BranchAndBoundSolver

model = read_json_model("examples/mip/binary_knapsack.json")
solution = BranchAndBoundSolver().solve(model)
```

SILO also has checked-in MIP examples under:

```text
examples/mip/
```

The previous design note recommended exposing MIP solving through a separate CLI command:

```bash
silo mip-solve MODEL_PATH
```

instead of overloading:

```bash
silo solve MODEL_PATH --solver branch-and-bound
```

Reason: existing `silo solve --solver` means LP backend selection (`tableau` or `revised`), while branch-and-bound is a MIP algorithm that uses LP backends internally.

This task implements the first MIP CLI command.

## 6. CLI Interface Requirements

Add command:

```bash
silo mip-solve MODEL_PATH
```

Required positional argument:

```text
MODEL_PATH
```

Optional flags:

```text
--output / -o
--lp-solver tableau|revised
--node-limit N
```

### 6.1 `--lp-solver`

This selects the LP backend used internally by branch-and-bound.

Allowed values:

```text
tableau
revised
```

Default:

```text
tableau
```

Important: do not reuse `--solver` for MIP. `--solver` remains the LP solve command's backend selector.

### 6.2 `--node-limit`

Accept a nonnegative integer.

Default should match `BranchAndBoundSolver` default unless importing the default is clean.

If invalid, argparse should return exit code `2`.

### 6.3 `--output`

Same behavior as `silo solve --output`.

If provided, write solution JSON to file.

If not provided, print solution JSON to stdout.

## 8. MIP Solve Behavior

Implement helper such as:

```python
def _mip_solve(
    model_path: str,
    output_path: str | None,
    lp_solver_name: str,
    node_limit: int,
) -> int:
    ...
```

Behavior:

1. check model path exists;
2. read model with `read_json_model`;
3. create LP backend:
   - `tableau` -> `TableauSimplexSolver()`
   - `revised` -> `RevisedSimplexSolver()`
4. create `BranchAndBoundSolver(lp_solver=..., node_limit=node_limit)`;
5. solve model;
6. output solution JSON using existing `solution_to_json` / `write_solution_json`;
7. return exit code based on solution status.

------

## 9. Exit Code Requirements

Use the same convention as `silo solve`:

```text
0 = solution.status == optimal
1 = non-optimal status, read error, model error, or write error
2 = argparse usage error
```

Examples:

```bash
silo mip-solve examples/mip/binary_knapsack.json
```

returns `0`.

```bash
silo mip-solve examples/mip/infeasible_binary.json
```

returns `1` with solution JSON status `infeasible`.

Missing file returns `1`.

Invalid `--lp-solver` returns `2`.

Invalid `--node-limit` returns `2`.
