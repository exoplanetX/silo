## 1. Background

SILO now has a relatively complete Phase 4 presolve layer:

- `silo presolve MODEL_PATH`;
- `silo solve MODEL_PATH --presolve`;
- empty-row diagnostics;
- empty-column diagnostics;
- fixed-variable elimination;
- repeated-pass presolve;
- original-space slack recovery;
- scaling diagnostics;
- checked-in JSON examples for presolve recovery.

The current `examples/json/` directory includes general LP examples and presolve-specific examples. Before moving to Phase 5 MIP, the expected behavior of all public examples should be fixed in a regression checklist.

This task is a regression and documentation task. It should not add solver algorithms or presolve reductions.

## 7. Expected Behavior Matrix

Create a documented behavior matrix in:

```text
docs/phase4_regression_checklist.md
```

The matrix should include, for each JSON example:

1. default solve expected exit code and status;
2. tableau solve expected exit code and status;
3. revised solve expected exit code and status;
4. solve with `--presolve` expected exit code and status;
5. tableau solve with `--presolve` expected exit code and status;
6. revised solve with `--presolve` expected exit code and status;
7. `silo presolve` expected exit code and presolve status;
8. whether `silo compare` is expected to be meaningful.

### 7.1 General LP examples

For examples such as:

```text
production.json
ge_row.json
equality_row.json
```

Expected:

```text
solve without presolve: optimal
solve with presolve: optimal
tableau: optimal
revised: optimal
presolve: usually no_change or reduced depending on model
compare: meaningful and consistent
```

### 7.2 Infeasible example

For:

```text
infeasible.json
```

Expected:

```text
solve without presolve: infeasible, exit code 1
solve with presolve: infeasible, exit code 1
presolve: may be no_change if infeasibility is not detectable by current presolve
compare: meaningful and consistent
```

Important: Do not require presolve to detect all infeasibility. Presolve only detects the simple diagnostics currently implemented.

### 7.3 Presolve-only fixed-bound examples

For:

```text
fixed_var_recovery.json
repeated_empty_row.json
presolve_infeasible_after_fixed.json
```

Expected:

```text
solve without presolve: likely error because fixed/nonzero bounds are unsupported by simplex
solve with presolve: expected behavior from presolve recovery
presolve: reduced or infeasible depending on example
compare: not necessarily meaningful unless compare is explicitly designed to run without presolve and both backends agree on error
```

Do not change default solve behavior to make these examples solve without `--presolve`.

## 10. Tests Required

### 10.1 Every example has expectations

Scan:

```text
examples/json/*.json
```

Expected:

```text
every file name appears in EXAMPLE_EXPECTATIONS
```

### 10.2 Every expected example exists

For every key in `EXAMPLE_EXPECTATIONS`, assert the file exists.

### 10.3 Default solve matrix

For every example, run equivalent of:

```bash
silo solve examples/json/<example>
```

Check expected exit code and solution status.

For examples expected to error, ensure status is `"error"` and exit code is `1`.

### 10.4 Tableau solve matrix

For every example, run:

```bash
silo solve examples/json/<example> --solver tableau
```

Check expected exit code and status.

### 10.5 Revised solve matrix

For every example, run:

```bash
silo solve examples/json/<example> --solver revised
```

Check expected exit code and status.

### 10.6 Solve with presolve matrix

For every example, run:

```bash
silo solve examples/json/<example> --presolve
```

Check expected exit code and status.

### 10.7 Tableau solve with presolve matrix

Run:

```bash
silo solve examples/json/<example> --solver tableau --presolve
```

Check expected exit code and status.

### 10.8 Revised solve with presolve matrix

Run:

```bash
silo solve examples/json/<example> --solver revised --presolve
```

Check expected exit code and status.

### 10.9 Presolve diagnostics matrix

For every example, run:

```bash
silo presolve examples/json/<example>
```

Check:

```text
exit code = 0
presolve.status matches expectation
```

Remember: presolve command returns `0` even for presolve statuses `infeasible` and `unbounded`, as long as diagnostics are produced.

### 10.10 Compare matrix

For examples where compare is expected to be meaningful, run:

```bash
silo compare examples/json/<example>
```

Check expected exit code and consistency.

For presolve-only examples, either:

1. skip compare with a documented reason; or
2. test that compare returns a consistent error if that is current behavior.

Do not alter compare semantics in this task.
