## 7. Standard-Form Transformation Rules

Create a builder function such as:

```python
build_standard_form(model: Model) -> StandardFormProblem
```

Alternative names are acceptable if clear.

### 7.1 Row normalization

For each original constraint, normalize RHS to be nonnegative.

If:

```text
a_i x <= b_i
```

and `b_i < 0`, transform to:

```text
-a_i x >= -b_i
```

If:

```text
a_i x >= b_i
```

and `b_i < 0`, transform to:

```text
-a_i x <= -b_i
```

If:

```text
a_i x = b_i
```

and `b_i < 0`, transform to:

```text
-a_i x = -b_i
```

Use the existing tolerance convention:

```text
silo.utils.numerics.DEFAULT_TOLERANCE
```

Clean near-zero RHS and coefficients to `0.0`.

### 7.2 `<=` rows

For a normalized `<=` row:

```text
a_i x <= b_i
```

add a slack variable:

```text
a_i x + s_i = b_i
s_i >= 0
```

The slack variable should be part of the initial basis.

### 7.3 `>=` rows

For a normalized `>=` row:

```text
a_i x >= b_i
```

add a surplus variable and an artificial variable:

```text
a_i x - r_i + a_i_art = b_i
r_i >= 0
a_i_art >= 0
```

The artificial variable should be part of the initial basis.

### 7.4 `=` rows

For a normalized equality row:

```text
a_i x = b_i
```

add an artificial variable:

```text
a_i x + a_i_art = b_i
a_i_art >= 0
```

The artificial variable should be part of the initial basis.

```python
@dataclass(frozen=True)
class StandardColumn:
    name: str
    role: str
    original_name: str | None = None
    source_constraint: str | None = None
```

Allowed role values should be simple strings for now:


```python
    objective_coefficients: tuple[float, ...]
```

You may adjust field names if clearer, but the structure must preserve:

2. row names;
3. equality-form matrix;
4. RHS;
5. objective coefficients over transformed columns;
6. objective constant;
7. initial basis;
8. artificial columns;
9. mapping back to original variables and constraints.

Do not use mutable lists inside frozen public dataclasses unless there is a clear reason.

## 9. Basis Dataclass Requirements

Create:

```text
src/silo/lp/simplex/basis.py
```

Define a small `Basis` abstraction.

Recommended structure:

```python
@dataclass(frozen=True)
class Basis:
    basic_columns: tuple[int, ...]
    nonbasic_columns: tuple[int, ...]

    def validate(self, column_count: int, row_count: int) -> None:
        ...

    def is_basic(self, column_index: int) -> bool:
        ...

    def status_for_column(self, column_index: int) -> str:
        ...

    def pivot(self, leaving_row: int, entering_column: int) -> "Basis":
        ...
```

Required behavior:

1. `basic_columns` length must equal row count;
2. basic and nonbasic columns must be disjoint;
3. union of basic and nonbasic columns must cover all columns;
4. column indices must be within range;
5. duplicate basic or nonbasic indices are invalid;
6. ordering is deterministic and preserved;
7. `pivot()` returns a new `Basis` object, not in-place mutation;
8. `pivot()` replaces the basic column in `leaving_row` with `entering_column`;
9. the leaving basic column becomes nonbasic;
10. `nonbasic_columns` remains deterministic after pivot.

For the first implementation, use public status strings:

```text
basic
nonbasic_lower
```

Do not add finite-upper-bound statuses in this task.

## 12. Tests Required: Standard Form

Add:

```text
tests/unit/test_standard_form.py
```

Test at least the following.

### 12.1 Simple `<=` LP

Model:

```text
maximize 2x + 1
subject to x <= 4
           x >= 0
```

Expected standard form:

```text
columns = x, slack_capacity
row = [1.0, 1.0]
rhs = 4.0
initial basis = slack column
artificial_columns = empty
objective = [2.0, 0.0]
objective_constant = 1.0
```

### 12.2 `>=` row

Model:

```text
maximize x
subject to x >= 2
           x <= 5
           x >= 0
```

Expected:

```text
surplus and artificial columns exist for the >= row
slack column exists for the <= row
initial basis includes artificial for >= row and slack for <= row
artificial_columns includes the artificial column
```

### 12.3 Equality row

Model:

```text
maximize x + y
subject to x + y = 4
           x <= 3
           y <= 3
```

Expected:

```text
artificial column exists for equality row
slack columns exist for <= rows
initial basis includes artificial for equality row
```

### 12.4 Negative RHS normalization

Model:

```text
maximize x
subject to -x <= -2
           x <= 5
```

Expected first row becomes equivalent to:

```text
x >= 2
```

Therefore standard form should include:

```text
coefficient for x = 1.0
sense after normalization handled as >=
surplus column = -1.0
artificial column = 1.0
rhs = 2.0
```

### 12.5 Unsupported finite upper bound

A variable with finite upper bound should raise `ValueError`.

### 12.6 Unsupported nonzero lower bound

A variable with lower bound other than zero should raise `ValueError`.

### 12.7 Unsupported integer/binary variables

Integer and binary variables should raise `ValueError`.

### 12.8 Unsupported minimization

Minimization should raise `ValueError` unless you implement a tested conversion. Prefer raising `ValueError` in this task.
