## 1. Background

SILO currently has:

1. a dense tableau simplex solver with Phase I / Phase II;
2. a standard-form builder;
3. a `Basis` dataclass;
4. a first revised simplex solver for already feasible slack-basis LPs.

The current revised simplex solver is in:

```text
src/silo/lp/simplex/revised.py
```

It currently supports only standard-form problems with no artificial columns. If `build_standard_form(model)` returns artificial columns, the solver returns `SolverStatus.ERROR` with a message explaining that artificial columns are unsupported.

The current standard-form builder is in:

```text
src/silo/lp/simplex/standard_form.py
```

It already creates:

```text
slack columns for <= rows
surplus and artificial columns for >= rows
artificial columns for = rows
initial basis
artificial_columns
```

This task should extend the revised simplex solver so that artificial-column cases are handled through a revised-simplex Phase I procedure.

------

## 2. Goal

Add Phase I support to the revised simplex solver.

After this task, `RevisedSimplexSolver` should solve small continuous maximization LPs with:

```text
<= rows
>= rows
= rows
negative RHS rows after normalization
```

provided the model still satisfies the current supported variable assumptions:

```text
continuous variables only
x >= 0
no finite upper bounds
no nonzero lower bounds
maximization only
```

The solver should:

1. build standard form;
2. if no artificial columns exist, keep the current feasible-basis path;
3. if artificial columns exist, run Phase I using the standard-form initial artificial basis;
4. detect infeasibility when the Phase I artificial objective is nonzero beyond tolerance;
5. remove or disable artificial columns before Phase II;
6. restore the original objective;
7. run the existing primal revised simplex Phase II loop;
8. return the same public `Solution` fields as the feasible-basis revised solver.

## 6. Mathematical Scope

The revised simplex solver should support:

```text
maximize   c^T x + c0
subject to rows with <=, >=, =
           x >= 0
           variables continuous
           no finite upper bounds
           no nonzero lower bounds
```

The standard-form builder already transforms the model into:

```text
maximize   c^T z + c0
subject to A z = b
           z >= 0
```

where `z` may include:

```text
original variables
slack variables
surplus variables
artificial variables
```

This task should implement Phase I over this transformed standard-form problem.

## 7. Phase I Objective Convention

For Phase I, minimize the sum of artificial variables:

```text
min sum artificial variables
```

Since the revised simplex solver is currently a maximization solver, represent Phase I as:

```text
maximize - sum artificial variables
```

That means Phase I objective coefficients should be:

```text
0 for original variables
0 for slack variables
0 for surplus variables
-1 for artificial variables
```

Use the existing revised simplex maximization reduced-cost convention:

```text
reduced_cost_j = c_j - pi^T A_j
```

Optimality condition for maximization:

```text
reduced_cost_j <= tolerance for all nonbasic j
```

An entering column is a nonbasic column with:

```text
reduced_cost_j > tolerance
```

Document this convention in code comments or tests.

## 10. Detecting Infeasibility

After Phase I reaches optimality, compute the Phase I objective value.

Because Phase I maximizes:

```text
- sum artificial variables
```

the best feasible Phase I objective should be approximately:

```text
0
```

If the Phase I objective value is less than `-tolerance`, the original LP is infeasible.

Return:

```python
Solution(
    status=SolverStatus.INFEASIBLE,
    message="Revised simplex Phase I detected infeasibility."
)
```

or similar.

Be careful with sign conventions. Add tests that would fail if the sign is wrong.

## 11. Transition from Phase I to Phase II

After Phase I succeeds:

1. artificial variables should not be allowed to enter Phase II;
2. artificial variables should not appear in public `Solution` fields;
3. Phase II should optimize the original objective;
4. the Phase II basis should remain feasible.

Preferred approach:

```text
Phase I standard-form problem
  -> solve Phase I
  -> pivot artificial basic variables out where possible
  -> remove artificial columns
  -> build a reduced StandardFormProblem without artificial columns
  -> remap Basis indices
  -> run Phase II with original objective
```

If removing artificial columns is too large for this task, it is acceptable to keep artificial columns internally **only if**:

1. artificial columns are excluded from entering-column selection during Phase II;
2. public solution fields ignore them;
3. tests verify artificial variables do not affect Phase II results.

However, the preferred approach is to remove artificial columns because it makes Phase II cleaner and matches the tableau implementation.

------

## 12. Handling Artificial Variables Still Basic

After Phase I, some artificial variables may remain basic with value zero.

Handle this case carefully.

Recommended behavior:

1. For each basic artificial column, try to pivot it out using a non-artificial nonbasic column with a nonzero coefficient in that row.
2. If no such pivot exists and the row RHS is zero, the row may be redundant.
3. For this first implementation, if redundant-row removal is too complex, return a clear `NUMERICAL_ISSUE` or `ERROR` with a message explaining that the artificial basis could not be cleaned.
4. Add tests for common feasible cases where artificial variables can be pivoted out.

Do not silently keep positive artificial basic values after Phase I.
