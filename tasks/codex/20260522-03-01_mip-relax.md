## 1. Background

Phase 5A drafted the branch-and-bound design note in:

```text
notes/15_branch_and_bound_design.md
```

The next Phase 5B step is to implement only the MIP-to-LP relaxation builder. The current LP solvers accept continuous maximization LPs with nonnegative variables and no direct finite variable upper bounds. Therefore, binary and bounded integer upper bounds must be represented as ordinary linear rows in the LP relaxation.

This task is not a branch-and-bound implementation task.

## 2. Goal

Add a small, deterministic relaxation builder that converts the first supported MIP class into a continuous LP model accepted by the existing tableau and revised simplex layers.

The builder should:

1. preserve original variables as continuous relaxation variables;
2. preserve original constraints and objective coefficients;
3. convert binary upper bounds into explicit rows;
4. convert bounded nonnegative integer upper bounds into explicit rows;
5. append local branching constraints as explicit rows;
6. reject unsupported model features with clear errors;
7. avoid mutating the original model;
8. expose deterministic metadata for generated rows.

## 4. Required Behavior

Implement a function similar to:

```python
build_lp_relaxation(model, branching_constraints=())
```

It should return a structured result containing:

```text
relaxation model
generated bound row names
generated branching row names
```

The exact dataclass names may follow repository style.

### Supported variables

- Continuous variables: lower bound 0 and no finite upper bound.
- Binary variables: lower bound 0 and upper bound 1, relaxed to continuous variable plus `x <= 1`.
- Integer variables: lower bound 0 and finite upper bound, relaxed to continuous variable plus `x <= U`.

### Rejections

Reject:

- minimization objectives;
- continuous variables with finite upper bounds;
- variables with nonzero lower bounds;
- binary variables not using `[0, 1]` bounds;
- integer variables without finite upper bounds;
- branching constraints on unknown variables;
- branching constraints on continuous variables;
- unsupported branching constraint senses;
- user constraint names that collide with the reserved generated-row prefix.

### Branching constraints

Represent branching constraints as explicit rows:

```text
x_j <= floor(value)
x_j >= ceil(value)
```

This task only needs the row representation. It does not need branching-rule logic.

## 5. Tests

Add deterministic tests covering:

- binary variables relax to continuous variables with generated `x <= 1` rows;
- bounded integer variables relax to continuous variables with generated `x <= U` rows;
- original model is not mutated;
- original constraints and objective are preserved;
- local branching constraints become generated rows;
- generated row names are deterministic and reported in metadata;
- the relaxation can be passed to `build_standard_form`;
- unsupported minimization, nonzero lower bounds, continuous finite upper bounds, binary bad bounds, unbounded integers, branching on unknown variables, branching on continuous variables, invalid branching senses, and reserved-prefix row names are rejected.
