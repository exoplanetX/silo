## 1. Background

SILO currently has:

1. a JSON model reader;
2. a tableau simplex solver with Phase I / Phase II support;
3. solution diagnostics including primal values, slack values, reduced costs, and basis status;
4. a solution JSON writer;
5. a CLI skeleton where `silo solve` is still a placeholder.

The next step is to connect these pieces into a minimal but real command-line solve workflow.

Current desired user-facing command:

```bash
silo solve tests/fixtures/lp_small/production.json
```

Expected behavior:

1. read the JSON model;
2. solve it using the native tableau simplex solver;
3. print deterministic JSON solution output to stdout by default;
4. optionally write solution JSON to a file.

This task should not expand mathematical solver scope. It should only wire together existing model I/O, tableau solving, and solution output.

------

## 2. Goal

Implement the first working CLI solve workflow for SILO.

After this task, the following commands should work:

```bash
silo --help

0 = solve command completed and solver returned OPTIMAL

```

## 6. CLI Design Requirements

Refactor the CLI if needed, but keep it simple.

Preferred command shape:

```bash
silo solve MODEL_PATH [--output OUTPUT_PATH]
```

Required options:

```text
MODEL_PATH
```

Path to a JSON model file.

```text
--output / -o
```

Optional path to write solution JSON. If not provided, print solution JSON to stdout.

Optional but acceptable:

```text
--solver tableau
```

If implemented, only support `tableau` for now. Do not add fake solver choices.

Do not introduce a complex CLI framework such as Click or Typer. Use standard `argparse`.

## 7. JSON Output Requirements

The CLI should output deterministic solution JSON with at least:

```json
{
  "basis_status": {},
  "dual_values": {},
  "message": "...",
  "objective_value": 21.0,
  "primal_values": {},
  "reduced_costs": {},
  "slack_values": {},
  "status": "optimal"
}
```

Use the same fields and ordering convention as `write_solution_json()`.

If writing to stdout, print the JSON string followed by a newline.

If writing to a file, create parent directories only if this is simple and safe. Otherwise, let normal file errors surface with a clear message.

Recommended implementation:

1. add a helper such as `solution_to_dict(solution: Solution) -> dict[str, object]`;
2. add a helper such as `solution_to_json(solution: Solution) -> str`;
3. make `write_solution_json()` use the same helper;
4. make CLI stdout output use the same helper.

This avoids duplicating JSON serialization logic.

## 8. Error Handling Requirements

### 8.1 Invalid file path

If the model path does not exist:

- print a clear error message to stderr;
- return exit code `1`.

Example message:

```text
Error: model file not found: <path>
```

### 8.2 Invalid JSON or invalid model

If `read_json_model()` raises `ValueError`:

- print a clear error message to stderr;
- return exit code `1`.

Example message:

```text
Error: failed to read model: ...
```

### 8.3 Solver returns ERROR

If tableau solver returns `SolverStatus.ERROR`:

- output the solution JSON if possible;
- return exit code `1`.

### 8.4 Solver returns INFEASIBLE or UNBOUNDED

If solver returns `INFEASIBLE` or `UNBOUNDED`:

- output the solution JSON;
- return exit code `1`.

### 8.5 Solver returns OPTIMAL

If solver returns `OPTIMAL`:

- output the solution JSON;
- return exit code `0`.
