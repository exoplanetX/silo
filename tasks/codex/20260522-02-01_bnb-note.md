### 7.2 Current LP and Presolve Boundary

Summarize current LP capabilities:

```text
continuous maximization LPs
tableau and revised backends
<=, >=, = rows
nonnegative variables
no direct finite upper-bound simplex support
presolve can eliminate fixed variables
solve-time presolve is opt-in
```

Discuss the critical issue:

```text
MIP needs binary/integer variables with bounds.
Current LP solvers do not directly accept integer/binary variable types or finite variable upper bounds.
```

Therefore, the MIP layer must construct a continuous LP relaxation that respects the current LP solver boundary.

------

### 7.3 First Supported MIP Class

Define the first supported MIP class conservatively.

Recommended first class:

```text
maximize c^T x + c0
subject to linear rows with <=, >=, =
           continuous variables x_c >= 0 with no finite upper bounds
           binary variables x_b in {0,1}
           bounded nonnegative integer variables x_i in {0,1,...,U_i}
```

But because current LP solvers do not support finite upper bounds directly, explain that finite bounds should be converted into explicit linear constraints in the LP relaxation:

```text
x_b <= 1
x_i <= U_i
```

All integer variables should have lower bound 0 in the first implementation.

Do not support:

```text
minimization
negative lower bounds
general finite lower bounds
semi-continuous variables
SOS constraints
indicator constraints
quadratic constraints
```

------

### 7.4 LP Relaxation Construction

Specify how a MIP model becomes an LP relaxation model.

For each variable:

#### Continuous variable

If continuous with lower bound 0 and no finite upper bound:

```text
keep as continuous variable
```

#### Binary variable

Transform to a continuous relaxation variable:

```text
0 <= x <= 1
```

Because LP solvers do not support upper bounds directly, represent upper bound as an added row:

```text
x <= 1
```

and use a continuous variable with lower bound 0 and no finite upper bound.

#### Integer variable

For nonnegative bounded integer variable:

```text
0 <= x <= U
```

Use explicit row:

```text
x <= U
```

for the LP relaxation.

If an integer variable has no finite upper bound, the first B&B solver may reject it unless branching constraints and problem rows provide effective bounds. Prefer requiring finite upper bound for integer variables in the first implementation.

#### Branching constraints

At a B&B node, add local branching rows:

```text
x_j <= floor(value)
x_j >= ceil(value)
```

For binary variables, branching becomes:

```text
x_j <= 0
x_j >= 1
```

These rows should be added to the LP relaxation as normal linear constraints.

### 7.6 Branching Rule

Choose a deterministic first branching rule.

Recommended:

```text
first fractional integer variable in model variable order
```

Fractionality test:

```text
abs(x_j - round(x_j)) > integer_tolerance
```

Use a dedicated integer tolerance, maybe:

```text
DEFAULT_INTEGER_TOLERANCE = 1e-6
```

but do not implement it in this design task.

For a fractional value `v`:

```text
left child:  x_j <= floor(v)
right child: x_j >= ceil(v)
```

Tie breaking:

```text
model variable order
```

No strong branching.

No pseudo-cost branching.

No reliability branching.

### 7.8 Incumbent and Bounds

For maximization MIP:

```text
incumbent_value = best known feasible integer objective
global_upper_bound = best LP relaxation bound among open nodes
MIP gap = global_upper_bound - incumbent_value
```

If no incumbent exists:

```text
incumbent_value = -infinity
```

If no open nodes remain:

```text
global_upper_bound can be incumbent_value or None
```

Define pruning rules:

```text
LP relaxation infeasible -> prune
LP relaxation unbounded -> mark carefully; may imply MIP unbounded if no integer restriction bounds it
LP relaxation bound <= incumbent + tolerance -> prune by bound
LP relaxation solution integer-feasible -> update incumbent and prune
otherwise branch
```

Discuss how to treat LP `UNBOUNDED` in the first implementation. Recommended conservative rule:

```text
If an LP relaxation at a node is unbounded, return MIP status UNBOUNDED only if no integer branching can bound the improving direction is established; otherwise, treat as ERROR/UNBOUNDED with clear tests deferred.
```

For the first implementation, avoid unbounded MIP cases as primary fixtures.
