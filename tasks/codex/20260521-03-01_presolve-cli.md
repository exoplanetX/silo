## 1. Background

SILO now has a conservative presolve layer with:

- immutable `PresolveResult`;
- `PresolveDiagnostics`;
- `ReductionRecord`;
- `ScalingDiagnostics`;
- empty-row diagnostics and feasible empty-row removal;
- empty-column diagnostics and simple unboundedness detection;
- fixed-variable elimination with solution recovery;
- coefficient-range and scaling diagnostics.

The CLI currently supports:

```bash
silo solve MODEL_PATH
silo solve MODEL_PATH --solver tableau
silo solve MODEL_PATH --solver revised
silo compare MODEL_PATH
```

The current CLI command choices are `compare`, `help`, and `solve`.

The next Phase 4 task is to expose presolve/scaling diagnostics through the CLI without connecting presolve to the default solve workflow.

------

## 2. Goal

Add a developer-facing CLI command:

```bash
silo presolve MODEL_PATH
```

The command should:

1. read a JSON model;
2. run `Presolver().run(model)`;
3. output deterministic JSON diagnostics to stdout by default;
4. optionally write diagnostics JSON to a file;
5. not run tableau or revised simplex;
6. not change `silo solve` behavior;
7. not change `silo compare` behavior;
8. not change JSON model schema or solution JSON schema.

After this task, the following commands should work:

```bash
silo presolve examples/json/production.json
silo presolve examples/json/production.json --output outputs/production_presolve.json
python -m silo.cli.main presolve examples/json/production.json
```

## 6. CLI Interface Requirements

Add a new command:

```bash
silo presolve MODEL_PATH
```

Required positional argument:

```text
MODEL_PATH
```

Optional argument:

```text
--output / -o
```

If `--output` is provided, write diagnostics JSON to that path.

If `--output` is not provided, print diagnostics JSON to stdout.

Do not add solver selection to this command.

Do not run tableau or revised simplex inside this command.

------

## 7. Exit Code Requirements

Use this exit-code convention:

```text
0 = model was read and presolve diagnostics were produced
1 = missing file, invalid JSON/model, or output write error
2 = argparse usage error
```

Important: `PresolveStatus.INFEASIBLE` or `PresolveStatus.UNBOUNDED` should still return exit code `0` for the `presolve` command, because the diagnostic command completed successfully and reported the status in JSON.

This differs from `silo solve`, where non-optimal solver statuses return exit code `1`.

Document this difference in `docs/presolve_cli.md`.

## 8. Presolve JSON Schema

The command should produce deterministic JSON with this general structure:

```json
{
  "model_path": "examples/json/production.json",
  "presolve": {
    "status": "no_change",
    "changed": false,
    "message": "No presolve reductions were applied.",
    "removed_rows": [],
    "removed_variables": [],
    "fixed_variables": [],
    "warnings": [],
    "notes": []
  },
  "reductions": [],
  "scaling": {
    "max_abs_coefficient": 3.0,
    "min_abs_nonzero_coefficient": 1.0,
    "coefficient_ratio": 3.0,
    "max_abs_rhs": 12.0,
    "max_abs_objective": 5.0,
    "warnings": []
  }
}
```

Exact numeric values depend on model.

### 8.1 Required top-level fields

Include:

```text
model_path
presolve
reductions
scaling
```

### 8.2 Required presolve fields

Include:

```text
status
changed
message
removed_rows
removed_variables
fixed_variables
warnings
notes
```

### 8.3 Required reduction fields

Each reduction should serialize:

```text
type
target
description
data
```

`data` should be deterministic.

If `ReductionRecord.data` is a tuple of key-value pairs, output it as a JSON object with deterministic key order if possible.

### 8.4 Required warning fields

Each warning should serialize:

```text
code
message
source
```

### 8.5 Required scaling fields

Include:

```text
max_abs_coefficient
min_abs_nonzero_coefficient
coefficient_ratio
max_abs_rhs
max_abs_objective
warnings
```

Do not include transformed model details in the first version.

Do not serialize full presolved model JSON in this task.

## 10. CLI Error Handling

### 10.1 Missing model path

If the model path does not exist:

```text
Error: model file not found: <path>
```

Return exit code `1`.

### 10.2 Invalid JSON or invalid model

If `read_json_model()` raises `ValueError`:

```text
Error: failed to read model: ...
```

Return exit code `1`.

### 10.3 Output write error

If writing output fails:

```text
Error: failed to write presolve diagnostics: ...
```

Return exit code `1`.

Keep stderr messages clear and consistent with existing CLI style.
