## 7. Content Requirements

### 7.1 Purpose

Explain why presolve and scaling are needed after tableau and revised simplex.

The note should connect presolve to:

```text
model simplification
early infeasibility detection
numerical diagnostics
safe preparation for future MIP
traceable transformations
```

Do not present presolve as a performance trick only.

### 7.2 Current Solver Boundary

Summarize current SILO status:

```text
tableau simplex backend
revised simplex backend
JSON model input
CLI solve
CLI compare
continuous maximization LPs
nonnegative variables
no finite upper bounds
no nonzero lower bounds
no MIP yet
```

Explain that early presolve must respect these boundaries.

### 7.3 Presolve Philosophy

The first presolve layer should be conservative.

Emphasize:

```text
deterministic transformations
small readable rules
explicit transformation records
reconstruction of original-space solution
no aggressive reductions
no hidden convention changes
```

State that a presolve step should be skipped if it cannot be tested and reversed clearly.

### 7.4 Transformation Traceability

Define the need for a future transformation log.

Suggested concept:

```python
@dataclass(frozen=True)
class PresolveResult:
    model: Model
    reductions: tuple[Reduction, ...]
    diagnostics: PresolveDiagnostics
```

Suggested concept:

```python
class Reduction:
    def apply(...)
    def recover_solution(...)
```

Do not implement these classes in this task, but describe the intended behavior.

Explain that each reduction should record enough information to reconstruct original variable values and interpret original constraints.

### 7.5 First Presolve Passes

Specify the first safe presolve passes.

Recommended Phase 4B scope:

```text
empty row detection
empty column detection
fixed variable substitution
simple bound validation
simple singleton-row bound tightening
duplicate row warning or diagnostic only
coefficient range diagnostics
```

For each, describe:

1. purpose;
2. input pattern;
3. transformation or diagnostic result;
4. required reconstruction information;
5. tests needed.

#### Empty rows

Cases:

```text
0 <= rhs
0 = rhs
0 >= rhs
```

and infeasible rows such as:

```text
0 <= -1
0 = 1
0 >= 1
```

Explain how these should be handled.

#### Empty columns

A variable not appearing in constraints can imply unboundedness depending on objective coefficient and bounds. Since current variables are nonnegative and have no upper bounds, this must be handled carefully.

Do not over-implement in the first presolve task.

#### Fixed variables

If a variable has `lower == upper`, substitute it into constraints and objective, then record the fixed value for solution reconstruction.

Note: current solver does not support nonzero lower bounds widely, but the model layer has bounds. Design the concept cautiously.

#### Singleton rows

For a row with one variable, simple bound tightening may be possible.

Example:

```text
a x <= b
x >= 0
```

can imply an upper bound if `a > 0`.

Since finite upper bounds are currently unsupported by solvers, this may remain diagnostic or future work unless the solver scope changes.

### 7.6 Solution Reconstruction

Explain how presolve should map solver-space solutions back to original variables.

Examples:

```text
fixed variable values restored
removed rows reported as satisfied or redundant
substituted objective constants restored
slack values computed from original constraints
```

Emphasize that public `Solution` must stay in original model space.

### 7.7 Scaling Diagnostics

Scaling should initially be diagnostic only.

Define diagnostics such as:

```text
max coefficient magnitude
min nonzero coefficient magnitude
coefficient ratio
large RHS values
large objective coefficient range
near-zero coefficients
```

Recommended initial output:

```python
@dataclass(frozen=True)
class ScalingDiagnostics:
    max_abs_coefficient: float
    min_abs_nonzero_coefficient: float | None
    coefficient_ratio: float | None
    warnings: tuple[str, ...]
```

Do not implement automatic scaling in the first implementation.

Explain why automatic scaling requires careful mapping of primal values, dual values, reduced costs, and objective values.

### 7.8 Numerical Diagnostics

Explain useful diagnostics for the current dense solvers:

```text
coefficient range
basis condition proxy
near-zero pivots
iteration limit
primal feasibility residual
constraint activity residual
objective reconstruction check
```

Do not require exact condition-number computations in the first implementation.

### 7.9 Module Layout

Recommend future files:

```text
src/silo/presolve/presolver.py
src/silo/presolve/reductions.py
src/silo/presolve/fixed_variable.py
src/silo/presolve/row_reduction.py
src/silo/presolve/bound_tightening.py
src/silo/presolve/scaling.py
src/silo/presolve/diagnostics.py
tests/unit/test_presolve.py
tests/unit/test_scaling_diagnostics.py
```

Explain the role of each module.

### 7.10 Testing Strategy

Specify tests for future implementation:

```text
no-op presolve leaves model unchanged
empty feasible row removal
empty infeasible row detection
fixed variable substitution
objective constant update after fixed variable substitution
solution reconstruction after fixed variable elimination
simple singleton-row diagnostics
coefficient range diagnostics
near-zero coefficient diagnostics
presolve result remains deterministic
original-space slack values remain correct
tableau/revised comparison still works after presolve is enabled later
```

### 7.11 Implementation Phases

Break Phase 4 into smaller implementation tasks:

```text
Phase 4A: presolve and scaling design note
Phase 4B: PresolveResult and diagnostics dataclasses
Phase 4C: empty-row and empty-column diagnostics
Phase 4D: fixed-variable elimination with solution reconstruction
Phase 4E: coefficient-range and scaling diagnostics
Phase 4F: optional CLI flag for presolve diagnostics
```

Do not implement these phases in this task; just define them.

### 7.12 Out of Scope

Clearly state that the first presolve implementation should not include:

```text
aggressive row aggregation
general redundancy detection
dominance detection
probing
dual reductions
MIP presolve
automatic scaling transformations
basis crash
bound-shifted solver support
finite upper-bound simplex support
dual-value rescaling
external solver comparison
performance benchmarking
```
