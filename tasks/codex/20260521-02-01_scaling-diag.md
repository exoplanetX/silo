## 1. Background

SILO currently has a conservative presolve layer with:

- immutable `PresolveResult`;
- `PresolveDiagnostics`;
- `ReductionRecord`;
- empty-row diagnostics and feasible empty-row removal;
- empty-column diagnostics and simple unboundedness detection;
- fixed-variable elimination with solution recovery.

The current `ScalingDiagnostics` dataclass exists, but scaling is still a placeholder. `Presolver.run()` currently stores `empty_scaling_diagnostics()` in every `PresolveResult`.

This task implements diagnostic-only coefficient-range and scaling diagnostics.

It should not automatically scale the model.

## 8. Coefficient Range Rules

### 8.1 Constraint coefficients

Collect absolute values of all nonzero constraint coefficients:

```text
abs(a_ij) > DEFAULT_TOLERANCE
```

Use:

```text
silo.utils.numerics.DEFAULT_TOLERANCE
```

For constraint coefficients:

```text
max_abs_coefficient = max(abs(a_ij))
min_abs_nonzero_coefficient = min(abs(a_ij))
coefficient_ratio = max_abs_coefficient / min_abs_nonzero_coefficient
```

If there are no nonzero constraint coefficients:

```text
max_abs_coefficient = 0.0
min_abs_nonzero_coefficient = None
coefficient_ratio = None
```

### 8.2 Near-zero coefficients

If a coefficient is nonzero but:

```text
0 < abs(a_ij) <= DEFAULT_TOLERANCE
```

record a warning.

Recommended warning:

```python
PresolveWarning(
    code="near_zero_coefficient",
    message="Coefficient is nonzero but within numerical tolerance.",
    source="<constraint_name>:<variable_name>",
)
```

Near-zero coefficients should not be counted as nonzero for min/max range.

Do not remove near-zero coefficients in this task.

### 8.3 RHS values

Compute:

```text
max_abs_rhs = max(abs(rhs_i))
```

If there are no constraints:

```text
max_abs_rhs = 0.0
```

### 8.4 Objective coefficients

Compute:

```text
max_abs_objective = max(abs(c_j))
```

over objective coefficients whose absolute values are greater than zero.

If objective coefficients are empty:

```text
max_abs_objective = 0.0
```

Near-zero objective coefficients may also produce warnings if they are nonzero and within tolerance.

Recommended warning code:

```text
near_zero_objective
```

Source:

```text
objective:<variable_name>
```

## 9. Warning Rules

Use deterministic warning order.

Recommended order:

1. near-zero constraint coefficient warnings in constraint order, then variable-name order as stored in the coefficient dictionary;
2. near-zero objective coefficient warnings in objective coefficient iteration order;
3. large coefficient-ratio warning;
4. large RHS warning;
5. large objective warning.

You may choose another deterministic order, but document and test it.

### 9.1 Large coefficient ratio warning

Add a warning when:

```text
coefficient_ratio is not None
coefficient_ratio >= 1e8
```

Recommended code:

```text
large_coefficient_ratio
```

Recommended message:

```text
Constraint coefficient range is large.
```

### 9.2 Large RHS warning

Add a warning when:

```text
max_abs_rhs >= 1e8
```

Recommended code:

```text
large_rhs
```

### 9.3 Large objective warning

Add a warning when:

```text
max_abs_objective >= 1e8
```

Recommended code:

```text
large_objective
```

Threshold constants may live in `scaling.py`:

```python
LARGE_COEFFICIENT_RATIO = 1e8
LARGE_RHS_MAGNITUDE = 1e8
LARGE_OBJECTIVE_MAGNITUDE = 1e8
```

If defined, test behavior around them.
