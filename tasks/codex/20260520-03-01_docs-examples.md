## 1. Background

SILO now has a working end-to-end solve workflow:

```text
JSON model file
  -> read_json_model()
  -> TableauSimplexSolver.solve()
  -> Solution
  -> deterministic JSON output through CLI
```

The current CLI supports:

```bash
silo solve MODEL_PATH
silo solve MODEL_PATH --output OUTPUT_PATH
silo solve MODEL_PATH -o OUTPUT_PATH
python -m silo.cli.main solve MODEL_PATH
```

However, the public documentation still needs to be updated. The README should no longer describe the CLI as only a placeholder. Users should be able to understand:

1. what SILO currently supports;
2. how to install it;
3. how to write a JSON model;
4. how to run `silo solve`;
5. how to interpret the solution JSON;
6. what limitations remain.

This task is documentation- and example-focused. It should not expand solver algorithms.

## 7. JSON Model Format Document

Create:

```text
docs/json_model_format.md
```

This document should explain the current JSON model schema.

It must include:

### 7.1 Top-level fields

```json
{
  "name": "production",
  "sense": "maximize",
  "variables": [],
  "objective": {},
  "constraints": []
}
```

### 7.2 Variables

Explain:

```json
{
  "name": "x1",
  "lower": 0.0,
  "upper": null,
  "type": "continuous"
}
```

Current supported values:

```text
type = continuous
lower = 0.0
upper = null
```

Mention that integer and binary variable types exist in the model layer, but the current tableau LP solver does not solve them.

### 7.3 Objective

Explain:

```json
"objective": {
  "coefficients": {"x1": 3.0, "x2": 5.0},
  "constant": 0.0
}
```

Mention that current tableau solve supports maximization.

### 7.4 Constraints

Explain supported row senses:

```text
<=
>=
=
```

Example:

```json
{
  "name": "labor",
  "coefficients": {"x1": 1.0, "x2": 2.0},
  "sense": "<=",
  "rhs": 8.0
}
```

### 7.5 Full examples

Include at least one complete example inline, preferably the production LP.

### 7.6 Current solver support warning

Clearly distinguish:

```text
The model format may represent more than the current tableau solver can solve.
```

For example, binary variables can be represented but are not solved by the current tableau LP solver.

## 10. User-Facing JSON Examples

Create directory:

```text
examples/json/
```

Add these small examples.

### 10.1 `examples/json/production.json`

A standard `<=` LP:

```text
maximize 3x1 + 5x2
s.t. x1 + 2x2 <= 8
     3x1 + 2x2 <= 12
     x1, x2 >= 0
```

Expected optimum:

```text
x1 = 2
x2 = 3
objective = 21
```

### 10.2 `examples/json/ge_row.json`

A model requiring Phase I through a `>=` row:

```text
maximize x
s.t. x >= 2
     x <= 5
     x >= 0
```

Expected optimum:

```text
x = 5
objective = 5
```

### 10.3 `examples/json/equality_row.json`

A model with equality row:

```text
maximize x + y
s.t. x + y = 4
     x <= 3
     y <= 3
     x, y >= 0
```

Expected objective:

```text
objective = 4
```

Do not require a unique split between x and y.

### 10.4 `examples/json/infeasible.json`

An infeasible model:

```text
maximize x
s.t. x >= 2
     x <= 1
     x >= 0
```

Expected status:

```text
infeasible
```

Keep all examples small and readable.
