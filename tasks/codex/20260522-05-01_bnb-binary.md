## 2. Goal

Implement a pure depth-first branch-and-bound solver for small binary maximization MIPs.

The first solver should:

1. support binary variables only among integer-restricted variables;
2. allow continuous variables only if they satisfy the current LP relaxation boundary;
3. build LP relaxations with `build_lp_relaxation()`;
4. solve LP relaxations using a selected native LP backend;
5. branch on the first fractional binary variable in model variable order;
6. use depth-first search;
7. maintain an incumbent;
8. prune infeasible nodes, bound-dominated nodes, and integer-feasible nodes;
9. return a public `Solution`;
10. expose a detailed branch-and-bound result object for tests.

Do not implement bounded general integer variables in this task.

## 9. Main Branch-and-Bound Logic

Use deterministic depth-first search.

High-level algorithm:

```text
1. Validate supported binary MIP class.
2. Create root node.
3. Initialize stack with root.
4. Initialize empty incumbent.
5. While stack not empty and node limit not reached:
   a. pop node depth-first
   b. build LP relaxation using node.branching_constraints
   c. solve LP relaxation
   d. prune if relaxation infeasible
   e. handle relaxation error/numerical issue
   f. prune if LP bound cannot beat incumbent
   g. if LP solution is integer-feasible:
        update incumbent
        prune
      else:
        choose first fractional binary variable
        create two children
        push children deterministically
6. If incumbent exists and all nodes processed/pruned:
      return OPTIMAL incumbent
7. If no incumbent and all nodes pruned infeasible:
      return INFEASIBLE
8. If node limit hit:
      return ITERATION_LIMIT with best incumbent if available
```

Keep the logic readable.

Do not optimize performance.

## 12. Branching and Child Order

For fractional value `v` of variable `x`:

```text
left:  x <= floor(v)
right: x >= ceil(v)
```

For binary variable:

```text
left:  x <= 0
right: x >= 1
```

Use existing:

```python
branch_on_value(x, v)
```

Child creation:

```text
left child gets first new node id
right child gets second new node id
```

Depth-first stack order should process the left child first.

Because stack pops last, push right child first, then left child.

Document this in tests.

## 13. Pruning Rules

Use pruning order:

```text
1. LP infeasible -> prune reason LP_INFEASIBLE
2. LP error/numerical issue -> stop with ERROR or NUMERICAL_ISSUE
3. LP unbounded -> return UNBOUNDED conservatively if encountered
4. Bound dominated -> prune reason BOUND_DOMINATED
5. Integer feasible -> update incumbent and prune reason INTEGER_FEASIBLE
6. Otherwise branch
```

For maximization bound pruning:

```text
if incumbent exists and lp_objective <= incumbent_value + tolerance:
    prune
```

Use a MIP tolerance such as:

```python
MIP_OBJECTIVE_TOLERANCE = 1e-9
```

Keep constants local to `branch_and_bound.py` unless reusable.

If search finishes with no incumbent and all processed nodes were infeasible or pruned:


