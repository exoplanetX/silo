## 1. Background

SILO now has:

- MIP-to-LP relaxation builder;
- MIP search dataclasses;
- binary branch-and-bound solver;
- bounded nonnegative integer branch-and-bound support.

The current branch-and-bound solver is available as a Python API, but there is no user-facing MIP CLI command yet.

Existing CLI commands remain LP-oriented:

```bash
silo solve MODEL_PATH
silo solve MODEL_PATH --solver tableau
silo solve MODEL_PATH --solver revised
silo compare MODEL_PATH
silo presolve MODEL_PATH
```

The next step should be to add checked-in MIP JSON examples and regression tests using the Python `BranchAndBoundSolver` API before exposing any CLI.

This task should not add MIP CLI behavior.

------

## 2. Goal

## 7. MIP Examples to Add

### 7.1 `binary_knapsack.json`

Model:

```text
maximize 6 item_1 + 10 item_2 + 12 item_3
subject to item_1 + 2 item_2 + 3 item_3 <= 5
item_1, item_2, item_3 binary
```

Expected optimum:

```text
item_1 = 0
item_2 = 1
item_3 = 1
objective = 22
```

This may mirror the existing test fixture, but it should live as a user-facing example under `examples/mip/`.

------

### 7.2 `binary_choice.json`

Model:

```text
maximize x + y
subject to x + y <= 1
x, y binary
```

Expected:

```text
objective = 1
one of x, y equals 1
the other equals 0
```

Do not require a specific selected variable unless branch-and-bound determinism makes it stable. If current solver is deterministic, document the expected stable solution in tests.

------

### 7.3 `integer_allocation.json`

Model:

```text
maximize 3x + 2y
subject to 2x + y <= 4
x integer in {0, 1, 2}
y integer in {0, 1, 2, 3}
```

Expected optimum:

```text
x = 1
y = 2
objective = 7
```

------

### 7.4 `mixed_binary_integer.json`

Model:

```text
maximize 5b + 3x
subject to 2b + x <= 4
b binary
x integer in {0, 1, 2, 3}
```

Expected optimum:

```text
b = 1
x = 2
objective = 11
```

------

### 7.5 `mixed_continuous_integer.json`

Model:

```text
maximize 5x + y
subject to x + y <= 3
x integer in {0, 1, 2}
y continuous >= 0
```

Expected optimum:

```text
x = 2
y = 1
objective = 11
```

------

### 7.6 `infeasible_binary.json`

Model:

```text
maximize x
subject to x >= 1
           x <= 0
x binary
```

Expected:

```text
status = infeasible
```

## 8. Regression Tests Required

Add:

```text
tests/regression/test_mip_json_examples.py
```

The tests should use the Python API, not CLI.

Required imports:

```python
from silo.io.json_reader import read_json_model
from silo.mip.branch_and_bound import BranchAndBoundSolver
from silo.lp.simplex.revised import RevisedSimplexSolver
from silo.core.status import SolverStatus
```

Use `TableauSimplexSolver` default or the default `BranchAndBoundSolver`.

### 8.1 Every MIP example has expectations

Scan:

```text
examples/mip/*.json
```

Assert every file has an expectation entry.

### 8.2 Every expectation file exists

For every expectation entry, assert the JSON file exists.

### 8.3 All examples are readable

For every example:

```python
read_json_model(path)
```

should succeed.

### 8.4 Solve all examples with default B&B

For each example, run:

```python
BranchAndBoundSolver().solve(model)
```

Expected statuses and objectives should match the expectation table.

### 8.5 Solve selected examples with revised LP backend

For at least these examples:

```text
binary_knapsack.json
integer_allocation.json
mixed_continuous_integer.json
```

run:

```python
BranchAndBoundSolver(lp_solver=RevisedSimplexSolver()).solve(model)
```

Expected objective should match.

### 8.6 Check primal values

For examples with unique expected solution, assert expected primal values.

Examples:

```text
binary_knapsack.json
integer_allocation.json
mixed_binary_integer.json
mixed_continuous_integer.json
```

Use `pytest.approx`.

### 8.7 Infeasible example

For:

```text
infeasible_binary.json
```

Expected:

```text

objective_value is None
primal_values == {}
```

### 8.8 Detailed result smoke test

For one example, use:

```python
details = BranchAndBoundSolver().solve_with_details(model)
```

Assert:

```text
details.nodes_processed > 0
details.nodes_created >= 1
details.log is not empty
details.solution.status == OPTIMAL
```
