## Objective

Add optional detailed node-log output for `silo mip-solve` behind an explicit `--node-log`
flag, while preserving the default compact solution JSON and the existing `--details`
summary diagnostics contract.

This task must use the existing branch-and-bound node log already available through
`BranchAndBoundSolver.solve_with_details()`. It must not change branch-and-bound search
logic, node ordering, pruning rules, statuses, or solution JSON semantics.

## Context

Phase 5J designed MIP diagnostics output in `notes/17_mip_diagnostics_output_design.md`.
Phase 5K implemented:

```bash
silo mip-solve MODEL_PATH --details
```

The current `--details` output is a wrapper with top-level `solution` and `diagnostics`
objects. The diagnostics object includes summary fields such as `node_count`,
`nodes_processed`, `incumbent_value`, `best_bound`, `relative_gap`,
`termination_reason`, `node_limit`, and `lp_solver`. It intentionally does not include a
node log.

The next narrow step is to expose detailed branch-and-bound node logs only when users ask
for them explicitly. The CLI should not start emitting node logs by default because that
would widen the public diagnostics contract and make output larger for ordinary solves.

## Required Behavior

Add a `--node-log` flag for `silo mip-solve`.

Behavior must be:

```text
silo mip-solve MODEL_PATH
```

Returns the existing compact solution JSON object with no diagnostics wrapper.

```text
silo mip-solve MODEL_PATH --details
```

Returns the existing `solution` plus summary `diagnostics` wrapper, with no `node_log`
field.

```text
silo mip-solve MODEL_PATH --details --node-log
```

Returns the same wrapper, but includes `diagnostics.node_log` as an array of node-log
entries.

Using `--node-log` without `--details` must be rejected as a command-line usage error with
exit code `2`. The error message should make clear that `--node-log` requires `--details`.

Each node-log entry must include only stable public diagnostic fields derived from the
existing node log:

```text
node_id
depth
lp_status
lp_objective
prune_reason
branching_variable
incumbent_value
message
```

The node log must not include LP tableaux, bases, full relaxation models, generated
constraint rows, primal vectors, dual vectors, or any other large internal data.

The `--output` flag must write the selected output mode:

- compact solution JSON when neither `--details` nor `--node-log` is present;
- summary wrapper JSON when `--details` is present without `--node-log`;
- wrapper JSON with `diagnostics.node_log` when both `--details` and `--node-log` are
  present.
