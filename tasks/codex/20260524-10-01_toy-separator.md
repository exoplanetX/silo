## Objective

Add one deterministic toy separator for a tiny fixture, with explicit validity
documentation and no performance claims.

## Context

The next Phase 6 design step is to add a deliberately tiny deterministic separator that
exercises the separator lifecycle without making branch-and-cut performance claims.

## Scope Lock

This task solves exactly one primary problem: implement and test one deterministic toy
separator under `src/silo/cuts/`.

The implementation must:

- add one toy separator that emits a single documented upper-bound cut only when a
  configured variable exceeds a configured bound by tolerance;
- document that this separator is valid only for fixtures where that upper bound is known
  to be globally valid outside the separator;
- keep emitted cuts as `CutCandidate` values;
- keep output deterministic;
- avoid LP relaxation mutation, branch-and-bound logic, cut materialization, and
  performance claims.
